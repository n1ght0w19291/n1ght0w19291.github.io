## ADDED Requirements

### Requirement: Blog schema supports optional parentPost field
The blog collection schema in `src/content.config.ts` SHALL include an optional `parentPost` field of type `z.string()` that stores the slug (Astro `id` last segment) of the parent post.

#### Scenario: Post without parentPost is a top-level post
- **WHEN** a blog entry has no `parentPost` frontmatter field
- **THEN** it is treated as a top-level post and appears in all listings

#### Scenario: Post with parentPost is a sub-post
- **WHEN** a blog entry has `parentPost: "picoctf-2024"` in frontmatter
- **THEN** it is treated as a sub-post of the post whose slug is `"picoctf-2024"`

#### Scenario: Schema validation rejects non-string parentPost
- **WHEN** a blog entry has `parentPost: 123` in frontmatter
- **THEN** Astro build fails with a schema validation error

### Requirement: Sub-post URLs are rooted under the parent
The URL for a sub-post SHALL follow the pattern `/posts/<parentPost>/<sub-slug>`, generated by the existing `getPath` utility reading the file's directory structure.

#### Scenario: Sub-post resolves to correct URL
- **WHEN** a sub-post lives at `src/data/blog/picoctf-2024/rsa-starter.md` with `parentPost: "picoctf-2024"`
- **THEN** its URL is `/posts/picoctf-2024/rsa-starter`

#### Scenario: Flat post URL is unchanged
- **WHEN** an existing flat post lives at `src/data/blog/my-post.md` with no `parentPost`
- **THEN** its URL remains `/posts/my-post`

### Requirement: Parent post page renders a child post index
When a post has sub-posts, the `PostDetails` layout SHALL render a children section listing the sub-posts sorted by `pubDatetime` ascending.

#### Scenario: Parent with children shows child list
- **WHEN** the rendered post has one or more sub-posts
- **THEN** the page includes a section listing each child post's title and link

#### Scenario: Parent with no children shows no child list
- **WHEN** the rendered post has zero sub-posts
- **THEN** no child list section is rendered

#### Scenario: Child list items link to sub-post URLs
- **WHEN** a parent post has a sub-post `rsa-starter` under `picoctf-2024`
- **THEN** the child list renders a link to `/posts/picoctf-2024/rsa-starter`

### Requirement: Build-time warning for dangling parentPost reference
During `getStaticPaths`, if a sub-post's `parentPost` value does not match any existing post id, the build SHALL log a warning to `console.warn`.

#### Scenario: Missing parent emits warning
- **WHEN** `parentPost: "nonexistent"` is set but no post with that slug exists
- **THEN** a `console.warn` message is emitted containing the sub-post's id and the missing parent slug

#### Scenario: Valid parent emits no warning
- **WHEN** `parentPost: "picoctf-2024"` is set and that post exists
- **THEN** no warning is emitted
